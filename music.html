Music Editor - Java

The Music Editor is a Java app which allows users to load, edit, and playback music they create. The user can expand and rescale the window, as well as scroll through the track via the arrow keys as well as the scrollbars. Additional features include being able to jump to the start or end of the song, pause and resume whenever necessary, and even add or remove notes at whim!

Below details the task, thought-process, and features of the project, as well as providing links to the code on Github.

The Task at Hand

Given as a task in my Object-Oriented Design course, the purpose of the project was to familiarize students with new software design concepts and patterns, particularly the Model-View-Controller (MVC) Pattern. The project consisted of multiple parts, in which we built the different aspects of the software application, improving upon at the previous iteration at each step.

Each student developed their own version of the program, and then was paired up with another to create the best version they possibly could. When I was paired with my partner we realized that my design pattern allowed for the components to be swapped out more easily, and the model was more intuitive for in terms of what a measure or note was. Thus, my version became the base of the project.

The Development Process

After the individual attempt and being paired up, the development process began to complete the final version of the project. Not knowing what the tasks down the road would be, my partner and I planned out the necessary parts of the project, as well as key details to remember. The most important detail that came out of the brainstorming session was making sure that each component was seperate, and able to be swapped out if need be (since again, the main purpose of the assignment was to utilize the MVC pattern).

The process as a whole went smoothly, with us splitting the work and bringing it together at various steps along the way. I was responsible for creating the various aspects of the View (both the auditory and visual representations), as well as handling the keyboard. My partner dealt with setting up communication between the Controller and Model, and we both shared responsibility in fixing any bugs found in previous steps of the cycle.

Final Results

After completing the multiple Views and setting up the Controller, my partner and I were able to link everything together, and the project was finally finished. At the end we produced a program which can dynamically load in notes from a file, read and play audio at real time, as well as allow for direct user control at any point. Additionally users could add notes directly inside the program, eliminating the need to have to load a new file each time.

However, we were surpised with one final task - having to switch Views with another team. Since our components were designed to do just that, we had no issue passing our View along to another team. The team which recieved our View had no issues with our code, and were able to complete that task easily. However, the View which we recieved was written poorly, and key aspects didn't allow for use to swap it in as is, one being a lack of separation from View and Controller. Once the other team reworked it with appropriate separations, it was able to fit in with out program spectacularly.
