Instagram PhotoTracker - Processing

Built in Processing, a programming language built on Java targeted toward designers, the Instagram PhotoTracker makes use of Instagram's API to display posts geographically around the downton Boston area. The program allows for the user to search based on hashtags, and each result links to the post on Instagram.

Below details the task, thought-process, and features of the project, as well as providing links to the code on Github.

The Task at Hand

The Instagram PhotoTracker is an application which came out of one of my interactive media courses, in which we were tasked with creating something that makes use of the programming skills learned in that class, as well as having a good user experience and design behind it.

Having taken many Computer Science courses in the past, and currently enrolled in Object-Oriented Design, the programming aspects of the project came together very easily. Thus, I had more time to figure out the design I wanted to go for, as well as how the user experience would tie in.

The Design Process

One of the first aspects of the project which I had to tackle was what exactly my application would focus on. The project allowed us to explore different technologies as well as different areas of focus - graphics, animation, games, data visualization, etc. I decided to focus on data visualization, focusing on the use of an open API. I decided to go this route since I wanted to create a project that seems engaging to most audiences, and since it had potential for future revisits if I ever wanted to improve upon it.

After deciding upon data-vis, I chose the Instagram API since it was one that I haven't worked with in the past and wanted to experiment with. Since I was working with Instagram, a very popular mobile app, I decided that my user experience would mimic that of their website. The opening screen of my app was influenced by the site, having the different posts explain how to use the application.

The Development Process

After having the idea planned out, it was time to start coding and bring the idea to life. As said previously, the majority of the coding was familiar to me, however I did reach a key point in the development process - deciding which enviornment to utilize. Processing is built on top of Java, and thus doesn't have all the features a web app would (such as built in input fields and scrolling). There was an alternative which was P5, a program to convert Processing code into P5 code, which is similar to Javascript. When trying to migrate to P5, my code didn't seem to work in it's current state, so I decided to stay in Processing, opting to take on the challenge of creating those custom components rather than having to go back and rewrite the code I had already written.

Creating an input field and scroll action did seem a bit challenging at first, however once I started I found the process interesting and actually fun! I had never had to think about the code that went into those components before, and building them myself was a unique experience to say the least.

Final Results

After creating the custom components and putting together the intro page, I had a working program which I was proud to display. While the program wasn't a web app like I intended it to be in the start, it had the same functionality and similar to feel to one. The extra time not spent trying to refactor code was put to use in optimizing the user experience, and that decision paid off.

The program itself allows the user to view different posts around the Boston area, and filter by the different tags associated with the post. The input field is a way add the new filters, and you can toggle between displaying all posts and the ones you currently have filtered through tags. I plan to revisit this project in the future, possibly creating a web app next time utilizing tools such as Node.js and Google APIs.
